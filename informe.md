# Introducción

La Programación Orientada a Objetos (POO) es un paradigma fundamental en el desarrollo moderno de software, ya que permite modelar sistemas mediante clases y objetos que representan entidades del mundo real.

El presente proyecto fue desarrollado con el propósito de aplicar conceptos fundamentales de POO mediante dos implementaciones principales:

1. Una plataforma web educativa interactiva sobre Programación Orientada a Objetos.
2. Un sistema funcional en Java para la gestión de un taller mecatrónico.

El desarrollo de ambos proyectos permitió fortalecer conocimientos relacionados con herencia, encapsulamiento, polimorfismo, interfaces, modularidad y manejo de excepciones.


# Objetivos

## Objetivo General

# Marco Teórico


## Programación Orientada a Objetos

La Programación Orientada a Objetos es un paradigma de programación basado en la creación de objetos que contienen atributos y métodos.


## Clase

Una clase es una plantilla que define atributos y comportamientos comunes.

## Objeto

Un objeto es una instancia de una clase.

## Encapsulamiento

Consiste en proteger los datos internos de una clase mediante modificadores de acceso.


## Herencia

Permite que una clase herede atributos y métodos de otra.

## Polimorfismo

Permite que un mismo método tenga diferentes comportamientos.


## Interfaces

Definen contratos que las clases deben implementar.

## Clases Abstractas

Son clases que no pueden instanciarse directamente y sirven como modelos generales.

## Excepciones

Mecanismo utilizado para controlar errores durante la ejecución.

- Registro de equipos
- Reparación de equipos

- Herencia
- Polimorfismo

- model
- service

- exceptions

- app
